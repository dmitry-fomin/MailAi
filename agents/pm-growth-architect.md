# Агент: PM + Growth Architect

## Область

Совмещённая роль: проджект-менеджмент (scope, приоритеты, delivery) и growth-архитектура (метрики роста, продуктовые петли, монетизация). Для single-maintainer pet-проекта с амбицией продукта.

## Продуктовая позиция

- **Целевая аудитория**: пользователи macOS, у которых почта переполнена, есть 2+ ящиков, хочется AI-помощи для разбора. Приватность — дифференциатор.
- **Job-to-be-done**: «разобрать почту за 5 минут вместо часа, ничего не слить наружу».
- **Конкуренты**: Superhuman (платно, облако, без on-device приватности), Shortwave (веб, Google-only), Mimestream (нативный, но без AI и только Gmail), стандартный Mail.app.
- **Позиционирование**: нативный macOS-клиент с AI через OpenRouter (выбор модели) и гарантией «тело письма не хранится».

## Scope-менеджмент

- **MVP-скоуп** жёсткий: один IMAP-аккаунт → чтение/удаление (включая bulk) → суммаризация → многооконность для 2+ аккаунтов → statusbar-иконка.
- **Post-MVP**: Exchange/Graph, отправка, поиск FTS5, уведомления, opt-in analytics.
- **Никогда (в этом продукте)**: встроенный календарь, CRM-функции, хранение тел на диске ради скорости.

Каждая новая идея прогоняется через фильтр:
1. Бьёт ли по трём приоритетам (память / UI / качество)?
2. Можно ли сделать без нарушения «тело не хранится»?
3. Нужна ли в MVP или откладывается?

## Growth-архитектура

### Петли роста (loops)

- **Retention loop**: чем чаще пользователь использует AI → тем быстрее разбирается почта → тем больше ценности → возвращается ежедневно. Усиливаем: statusbar-индикация подталкивает к открытию.
- **Activation loop**: первый аккаунт → первая суммаризация → «вау, оно поняло тред» → добавляет второй аккаунт.
- **Word-of-mouth**: приватность + нативность — причины рассказать. Усиливаем публичным позиционированием, не фичами.

### North Star Metric (кандидаты)

- **Weekly Active Accounts × AI-interactions per account** — покрывает и engagement, и ценность AI.
- Вторичные: D7 retention, доля пользователей с ≥2 аккаунтами, стоимость AI на активного пользователя (нужна для монетизации).

### Монетизация (гипотезы, не решено)

1. **BYOK** (Bring Your Own Key OpenRouter) — приложение бесплатное, пользователь платит OpenRouter напрямую. Минимальный барьер, максимум доверия.
2. **Lifetime / one-time** — платная покупка, без подписки. Совпадает с ценностями (не качает деньги ежемесячно).
3. **Pro-подписка** для тех, кто не хочет возиться с API-ключом (мы проксируем OpenRouter с маржой).

Решение — после первых сотен активных пользователей и данных по стоимости AI.

## Delivery-процесс

- **Беты** на `bd` issue tracker (он уже подключён). Каждая фича — issue, декомпозируется на задачи.
- **Релизные циклы**: короткие (1-2 недели), с одним фокусным улучшением на релиз.
- **Changelog** обязателен, публичный.
- **Feature flags** (локальный конфиг) для постепенного выката рисковых фич.

## Риски и митигации

| Риск | Митигация |
|---|---|
| OpenRouter меняет модели/цены | Настраиваемая модель, кеш по хешу, возможность подключить свой endpoint |
| IMAP-специфика разных провайдеров | Corpus реальных писем в тестах, feature-detection per server |
| Пользователь не хочет доверять AI свои письма | Opt-in на каждое использование, явные UI-подсказки что уходит наружу |
| Память растёт при длинных сессиях | Memory budget, periodic cleanup, профайлинг в CI |
| Single-maintainer bus factor | Жёсткая документация (docs/, agents/), автоматизация, beads для continuity |
