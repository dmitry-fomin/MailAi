# Агент: AI / OpenRouter

## Область

Интеграция с OpenRouter: клиент, промпты, сценарии (суммаризация, оценка важности, предложения для массового удаления).

## Клиент

- **URLSession + async/await**, без сторонних SDK.
- Endpoint: `https://openrouter.ai/api/v1/chat/completions`.
- Headers: `Authorization: Bearer <key>`, `HTTP-Referer: mailai://local`, `X-Title: MailAi`.
- Модель — настраивается пользователем, дефолт — из конфига (обновляем по мере появления новых дешёвых моделей).
- **Streaming** (`"stream": true`) — обязательно, парсим SSE, отдаём чанки в UI.

## Промпты

- Шаблоны — в отдельных файлах `Prompts/*.txt`, версионируются.
- Плейсхолдеры: `{{subject}}`, `{{from}}`, `{{body}}` и т.д. — подставляются безопасно (не позволяем injection через тему письма — экранируем).
- System message — жёсткий: «ты ассистент почты, отвечай кратко, русский/английский по языку письма, никогда не придумывай факты».
- Few-shot примеры — в system message, чтобы экономить на входных токенах при частых вызовах.

## Экономия токенов

- **Кеш по хешу тела** — хеш (SHA-256) записывается в `messages.summary_hash`. Повторный запрос с тем же хешем — отдаём сохранённую summary (summary хранится, тело — нет).
- Для оценки важности — отправляем только заголовки (from, subject, snippet из темы), не тело.
- Для суммаризации треда — сначала извлекаем quoted parts и подписи, отправляем только свежий контент.
- Ограничение на размер входа: 100k токенов, тело режется по границам параграфов.

## Приватность

- Тело письма отправляется в OpenRouter **только** по явному действию пользователя.
- После получения ответа — тело из памяти освобождается (`body = nil`).
- В логах — только: модель, входные/выходные токены, длительность, код ошибки. **Никогда** — содержимое промпта и ответа.

## Retry / rate limiting

- 429 и 5xx — экспоненциальный backoff (1s, 2s, 4s, 8s, max 60s).
- Уважаем `Retry-After` header.
- Пользовательская отмена — приоритетнее retry.

## Массовое удаление (критичный сценарий)

- Двухшаговое: (1) AI предлагает кандидатов по запросу, (2) пользователь подтверждает.
- Запрос: «удали все рассылки за полгода» → AI возвращает список `message_id`, UI показывает их и предлагает подтвердить.
- **Никогда** не удаляем без подтверждения. Никаких auto-apply.
