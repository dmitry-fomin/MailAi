# Агент: Web Analytics

## Область

Аналитика продукта: метрики использования, воронки, retention, когортный анализ. Для десктоп-приложения — с поправкой на приватность и отсутствие веба.

## Базовый принцип

MailAi — приложение о приватности. Аналитика **по умолчанию выключена**. Пользователь явно опт-инится («помочь улучшить MailAi»). Без его согласия никакие события за пределы устройства не уходят.

## Что измеряем (при opt-in)

- **Активация**: первый запуск → добавлен первый аккаунт → прочитано первое письмо → использована первая AI-функция.
- **Engagement**: DAU/WAU/MAU окон-аккаунтов, средняя длина сессии, количество открытых писем.
- **AI-фичи**: частота использования суммаризации, bulk-delete, оценки важности. Success rate (пользователь применил предложение vs отменил).
- **Ошибки**: коды ошибок транспорта/AI, частота crash-ов (через отдельный opt-in).
- **Производительность**: время до первого письма в списке, latency AI-ответа, расход памяти.

## Что НЕ измеряем никогда

- Содержимое писем, адреса, темы.
- Имена mailbox'ов кроме стандартных ролей (INBOX, Sent и т.п.).
- Конкретные запросы пользователя в поиск / AI.
- Идентификаторы, позволяющие связать пользователя с его почтой (email-адрес, UID, message-id).

## Технически

- Идентификатор — случайный UUID на устройство, хранится в Keychain, сбрасывается при «reset analytics».
- Событие = `{ name, timestamp, duration?, success?, error_code? }`. Никаких free-text полей.
- Батчинг: отправка раз в N минут / при закрытии, по HTTPS.
- Backend — TBD (собственный endpoint или self-hosted Plausible/PostHog). **Не** Google Analytics, не Mixpanel — не проходят требованиям приватности.

## Воронки и когорты

- Активация: install → account added → first_read → first_ai_use.
- Retention по когортам недели установки.
- A/B-тесты — **не** на MVP. Позже — feature flags через локальный конфиг + per-user rollout по хешу UUID.

## Дашборды

- Основные метрики: DAU, активация, AI usage, top errors.
- Alert на рост crash-rate, падение активации.

## Запрещено

- Трекать без явного opt-in.
- Отправлять сырые строки из UI (только enum'ы событий).
- Third-party SDK, собирающие device fingerprint.
- Связывать analytics-id с любыми персональными данными.
