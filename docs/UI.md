## Назначение

Переиспользуемые SwiftUI- и AppKit-компоненты без бизнес-логики. То, что могло бы быть сторонней UI-библиотекой — но мы пишем сами.

## Ключевые сущности

- `MessageListView` — виртуализованный список писем (на 100 000+ элементов). Под капотом — `NSTableView` через `NSViewRepresentable`, если SwiftUI `List` проседает.
- `MessageReaderView` — чтение письма, стриминговый рендер тела.
- `SidebarView` — список папок аккаунта.
- `ImportanceBadge` — цветовая индикация важности (синхронизирована со `StatusBar`).
- `EmptyStateView`, `LoadingView`, `ErrorView` — стандартные состояния.
- `ToolbarButtons` — унифицированные кнопки toolbar.

## Принципы

- Компоненты **не знают** про транспорт/БД/AI. Получают данные через биндинги и колбэки.
- Все views `@MainActor`. Тяжёлые вычисления — во view model.
- Предпочтительно SwiftUI; AppKit — только там, где SwiftUI не справляется (виртуализация больших списков, продвинутая клавиатурная навигация).

## Зависимости

- **От**: `Core` (только модели для type-erased биндингов).
- **Кто зависит**: `AppShell`, `Search`, `Attachments`.
