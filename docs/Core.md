## Назначение

Доменный фундамент приложения. Типы, протоколы и ошибки, от которых зависят все остальные модули. Не содержит ни UI, ни сетевых вызовов, ни I/O.

## Ключевые сущности

- `Account` — описание подключённого ящика (тип: IMAP/Exchange, email, displayName, идентификатор в Keychain). **Без пароля.**
- `Mailbox` — папка (INBOX, Sent, Trash, кастомные). Иерархия, флаги, счётчики.
- `Message` — метаданные письма: `messageId`, `uid`, `from`, `to`, `subject`, `date`, `flags`, `size`, `hasAttachments`, `importance` (AI-оценка, опционально).
- `MessageBody` — **in-memory only** тело письма с вложениями. Никогда не сериализуется в хранилище.
- `Thread` — связка сообщений по `References`/`In-Reply-To`.
- `Attachment` — метаданные вложения (имя, mime, размер). Содержимое — стрим, не значение.
- `MailError` — унифицированный enum ошибок транспорта/хранилища/AI.

## Протоколы (границы модулей)

- `MailTransportProtocol` — реализуется в `MailTransport`.
- `MetadataStoreProtocol` — реализуется в `Storage`.
- `AIProviderProtocol` — реализуется в `AI`.
- `SecretsProtocol` — реализуется в `Secrets`.

## Бизнес-логика

Нет. Этот модуль — типы и контракты. Любая логика — в доменных модулях.

## API (публичный)

Все типы выше + протоколы. Помечены `public`, соответствуют `Sendable` там, где пересекают границы акторов.

## Зависимости

- **От кого зависит**: Foundation.
- **Кто зависит**: все остальные модули проекта.
